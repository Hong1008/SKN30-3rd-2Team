# [재설계 N] 독소 탐지 recall 붕괴 — 코퍼스 당사자 어휘 편중

> 근거: [07-03 결정 로그](../dicision/07-03.md) §4(실측 규명).
> 선행: 없음. **J·K·L·M(NONE/CHANGED 재설계)과 완전히 독립** — 별개 메커니즘(코퍼스 매칭)이라 언제든 착수 가능.

## 배경 (07-03 §4 실측 요약)

[toxic_patterns.json](../../data/03_normalized/toxic_patterns.json) 40건(8개념×5예문)의 당사자 어휘가
SW 프리랜서 어휘(도급인·수급인)로만 작성돼 있다 — 실측: 도급인 60회·수급인 55회, 원사업자·수급사업자
0회. 반면 SI/SM 골든의 독소 케이스는 전부 원사업자/수급사업자 어휘를 쓴다. 그 결과 독소 recall이
**SW 1.000 vs SI 0.429 vs SM 0.077** — 임계값 문제(v1_review §2 원인 B)와는 **별개의 2차 원인**이며,
임계값을 아무리 조정해도 해소되지 않는다(어휘가 안 맞으면 애초에 후보로 안 뜬다).

## 문제 정의 (해결책 아님 — 착수 세션이 설계할 것)

리랭커는 독소 *개념*(예: 일방적 대금 지연·과도한 손해배상)이 같아도 **당사자 표현이 다르면**
점수를 낮게 준다 — "도급인은 지연 이자를 면제한다" vs "원사업자는 지연 이자를 면제한다"가
개념적으로 동일함에도 하도급 계약서 조항이 SW 어휘 코퍼스에 안 물린다.

## 검토할 방향 (07-03 §4에서 거론 — 순위·선택은 착수 세션 몫)

1. **매칭 전 당사자 용어 정규화**(원사업자↔도급인, 수급사업자↔수급인을 역할 토큰으로 치환) —
   코퍼스는 그대로 두고 검색 질의/코퍼스 양쪽을 정규화. 유형 무관하게 일반화되는 장점.
2. **코퍼스에 하도급 어휘 변형 추가**(40건 → ~80건, 개념당 SW/SI-SM 어휘 쌍으로 이중화).
3. 개념당 예문 확대(5 → 10+) — 검색 기반 탐지의 근본적 얇음 보완.

## 부가 관찰 (설계 시 참고)
- 독소 개념 8종 선정 자체는 타당(IP 무상귀속·과도 경업금지·대금 지연·일방 변경·과도 손배·일방 해석·
  일방 해지·무상 추가업무).
- `NONCOMPETE_EXCESS`가 `CONFIDENTIALITY` 카테고리에 배치돼 있음(경업금지≠비밀유지, 애매) — 분류
  체계 정리도 겸사겸사 검토 대상.
