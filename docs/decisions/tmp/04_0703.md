## 평가 데이터·진단

### D48. v2 골든셋 결함을 튜닝 문제와 분리
- **결정:** threshold 튜닝을 계속하지 않고 v3 골든셋을 조 전체 텍스트와 엄격한 NONE 라벨 기준으로 재제작한다.
- **이유:** v2 NONE의 80%가 다항 표준조항에 대한 단문이라 coverage 조건상 NONE이 될 수 없었다. 기존 리뷰의 경고·체크리스트도 미이행되었고 일부 라벨은 의미상 오매핑됐다.
- **상태:** 원인 확정, v3 제작이 필수 선행 작업 (07-03)

## 제품 경계·판정 아키텍처

### D49. 관련성 신호와 의미 동치성의 한계를 명시
- **결정:** 글자 일치율·리랭커 점수·규칙 게이트를 쌓아 NONE↔CHANGED 동치성을 해결하려는 접근의 한계를 인정한다.
- **이유:** 리랭커는 주제 관련성이지 의미 보존을 판정하지 않으며, 강한 말바꿈일수록 규칙 방어선의 정렬 가드가 꺼지는 역설도 확인됐다.
- **상태:** 분석 결론 (07-03)

### D50. 1차는 근거 패키지, 동치 판정은 서버 밖 2차로 이관 검토
- **결정 방향:** 1차 MCP는 검색·정렬·MISSING/NO_MATCH/EXTRA·규칙 플래그를 결정론적으로 제공하고, NONE↔CHANGED 의미 동치성은 데모/클라이언트의 2차 LLM이 판정하는 구조를 검토한다.
- **이유:** 서버가 최종 동치 라벨을 확정하는 현 구조가 RAG의 검색→생성 역할 분리를 잘라냈다는 진단이다.
- **미결:** 반환 계약의 의미 변경 승인, LLM 결과 검증 방식, 규칙 코드의 근거 플래그 재정의.
- **상태:** 명제 합의, 정식 설계·사인오프 대기 (07-03)

## 독소 코퍼스·검색 품질

### D51. 독소 recall 저하의 원인을 당사자 어휘 편중으로 규명
- **결정:** SI/SM 독소 recall 하락을 threshold 문제가 아니라 SW 프리랜서 어휘에 치우친 독소 코퍼스의 구조 문제로 분류한다.
- **근거:** 코퍼스는 도급인/수급인만 사용하고, SI/SM 골든은 원사업자/수급사업자만 사용해 계약 유형별 recall이 1.000, 0.429, 0.077로 갈렸다.
- **후속 방향:** 매칭 전 역할 토큰 정규화를 우선 검토하고, 필요 시 하도급 어휘 변형·예문 확대를 사전 승인 후 진행한다.
- **상태:** 원인 확정, 데이터/코드 변경 보류 (07-03)
