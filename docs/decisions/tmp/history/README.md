## 편집 원칙

- 초기 기획부터 법률 자문·유불리 판단은 비목표였다. “판정을 하려다 포기했다”라고 쓰지 않는다.
- 변화는 검색 점수와 규칙에 부여했던 의미를 좁히고 초기 제품 경계를 구체화한 과정으로 설명한다.
- 법률 내용을 해설하지 않는다. 전문 도메인을 선택하며 구현 난도와 책임 경계를 체감한 배경만 짧게 다룬다.
- 기각·보류 결정은 별도 실패 목록으로 분리하지 않고 각 편의 인과관계에 포함한다.
- 과거 설계와 현행 구조를 구분한다. 특히 `CHANGED`와 의미 coverage 2-pass는 현행 계약이 아니다.
- 수치는 우열 홍보가 아니라 당시 결정을 내린 근거로만 사용한다.
- 각 글은 문제 → 가설 → 실측 → 결정 → 다음 문제 순서로 진행한다.
