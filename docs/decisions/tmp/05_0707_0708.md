## 1차/2차 경계와 계약 재정의

### D52. 규칙 기반 의미 판정과 coverage 판정을 제거
- **결정:** 정규식 기반 치명변경 탐지·difflib 폴백을 제거한 뒤, 더 나아가 `CHANGED`와 서브청크 의미 coverage 2패스 자체를 폐지한다.
- **이유:** 한국어 표현 열거는 끝나지 않고, 마커 없는 의미 반전·당사자 역할 반전은 규칙으로 잡을 수 없다. 리랭커 관련성도 내용 차이를 판정하지 못한다.
- **새 1차 판정:** 최고 후보가 threshold 이상이면 잠정 NONE, 미달이면 EXTRA; MISSING/NO_MATCH는 유지한다.
- **상태:** 사용자 승인 후 구현 완료 (07-07 후속)

### D53. 2차 LLM은 서버 밖에서 비교 근거를 해석
- **결정:** MCP 서버에는 LLM을 넣지 않고, 서버 밖 클라이언트/데모가 잠정 NONE과 EXTRA의 비교 근거를 받아 최종 의미 판단을 수행한다.
- **이유:** 1차는 결정론적 후보·근거 제공에 집중하고, 의미 해석은 2차 역할로 분리한다. `DeviationResult` 필드를 늘리지 않고 기존 텍스트·후보·점수로 최소 변경을 택한다.
- **상태:** 경계 확정; K 구현은 미착수 (07-07)

### D54. EXTRA에도 최고 후보를 보존
- **결정:** `select_best_match`는 threshold와 무관하게 최고 후보를 반환하며, deviation 분류가 threshold로 EXTRA/NONE을 가른다. MISSING의 커버됨 집계는 NONE만 반영한다.
- **이유:** EXTRA도 2차가 설명할 비교 대상을 가져야 하며, 임계 미달 후보가 MISSING 판정을 오염시키지 않게 한다.
- **상태:** 채택 및 회귀 보강 (07-07 후속)

## 골든·평가 방식

### D55. 현실적인 v3 골든과 검증기를 우선
- **결정:** 조 전체 텍스트·reorder·EXTRA를 포함한 v3 골든을 만들고, 현재 코드에 맞춰 문구를 순화하지 않는다. 스키마 검증기로 데이터 계약을 검사한다.
- **이유:** 코드를 통과시키기 위한 골든은 실제 말바꿈·의미 반전 약점을 숨기는 과적합이 된다.
- **상태:** SW 파일럿 및 SI/SM 확장 확인; L 문서 갱신은 후속 (07-07~08)

## 법령 grounding·도메인 분류

### D56. Grounder를 계약유형 인지형으로 확장
- **결정:** `get_grounding(category, contract_type=None)`을 가법 확장하고, SI/SM은 하도급법 우선 override 후 공용 법령으로 fallback한다.
- **이유:** 기존 고정 매핑은 SI/SM 계약의 하도급 맥락과 최빈 `SUBCONTRACTING`을 반영하지 못했고, eval의 NullGrounder 때문에 숨은 결함이었다.
- **상태:** 구현 완료; 단일 조문이 없는 일부 범주는 공용 fallback 유지 (07-07 후속)

### D57. 계약유형별 Category 스코핑과 신규 범주
- **결정:** `Category.contract_types`로 normalize 후보를 계약유형별로 제한하고, `INDUSTRIAL_SAFETY`·`INFO_SECURITY`를 추가한다.
- **이유:** SI/SM 안전·보안 조항이 기존 범주에 무작위로 흩어졌고 근로계약 전용 범주가 하도급에 새고 있었다. 법령 실측으로 적용 범위를 검증했다.
- **상태:** 코드 반영; 기존 정규화 산출물 재생성은 대기 (07-07 후속)

## 매칭 신뢰성

### D58. 이중 sigmoid를 제거해 EXTRA/NONE 축퇴 수정
- **결정:** CrossEncoder가 이미 반환하는 0~1 확률에 sigmoid를 한 번 더 적용하던 로직을 제거한다.
- **이유:** 이중 적용이 점수를 0.5~0.731로 압축해 threshold 0.5에서 사실상 모든 조항을 NONE으로 만들었다.
- **검증 기록:** 실제 운영 모델 재평가에서 이탈 F1이 SI 0.903, SM 0.909, SW 1.000으로 회복했다.
- **상태:** 구현·재검증 완료 (07-08)

### D59. 강한 paraphrase와 독소 축퇴는 별도 문제로 유지
- **결정:** sigmoid 수정 뒤 남은 강한 말바꿈 오차는 리랭커의 실제 한계로, SI/SM 독소 TP=0은 독소 코퍼스 편향 문제로 분리한다.
- **이유:** 전자는 코드 축퇴가 아니라 의도적으로 현실적 문구를 사용한 골든이 드러낸 검색 한계이며, 후자는 매칭 임계값 변화에도 불변이었다.
- **상태:** N 카드로 원인 조사·개선 보류 (07-08)
