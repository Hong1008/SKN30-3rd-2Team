## 3. v5 이탈 평가 골든셋 설계·구현

### 설계 (`v5_design.md`)

- 전체 135건: SI/SM/SW 각 45건 (tuning 30 + held-out 15)

## 4. 독소 오류 원인 분류표 (`R_toxic_error_taxonomy.md`)

### 내용

v3/v4 독소 진단의 FP/FN 62건을 결정론적·재현 가능한 규칙으로 분류했다.

- `primary_reason`: `SEARCH_MISS` / `LOW_SCORE` / `OVER_MATCH` / `UNMAPPED_PATTERN`
- `semantic_dimension`: `none` / `negation` / `period` / `numeric` / `authority_subject` / `unmapped`
- 현재 62건: `SEARCH_MISS` 0건, `LOW_SCORE` 26건, `OVER_MATCH` 36건, `UNMAPPED_PATTERN` 0건

주요 관찰:

- FP(OVER_MATCH) 36건은 정상 보호조항과 독소 예문의 표면 주제가 같은 경우에 집중
- FN(LOW_SCORE) 26건 중 v4에서 `negation` 9건, `authority_subject` 7건이 관찰됨
- `period`와 `UNMAPPED_PATTERN`은 현 입력에서 0건 (부재 증명이 아닌 미확인)

### 결정

이 분류표는 개선안의 타겟 선택 근거로 사용하되, 코퍼스·임계값·런타임 코드를 변경하지 않는다.

## 6. 평가 CLI 격리 옵션

### 변경

`src/eval/run_eval.py`에 실험 격리용 인수를 추가했다:

- `--case-ids`: 실험 split을 특정 case ID로 제한. 골든에 없는 ID는 즉시 `ValueError`.
- `--no-write`: 진단·결과 파일을 디스크에 쓰지 않음.
- `--output-dir`: 산출물을 별도 디렉터리에 기록 (동결 골든 디렉터리 보호).
- `_select_case_ids()`: 골든 원래 순서 유지, 미지 ID 거부.

### 카드 8 미진행 상세

카드 8은 카드 7(실험 S 구현)의 후속으로, 다음 절차를 **순서대로** 수행해야 한다.

   - 통과 조건: FN ≤ 7, FP ≤ 5, F1 > 0.563
2. **tuning 통과 시에만** 동결된 v4 held-out (18건)을 **단 한 번** 실행.
   - 통과 조건: FN ≤ 6, FP ≤ 1, F1 > 0.364
3. 결과에 따라 `채택` / `보류` / `폐기` 중 하나를 명시.
4. 채택되더라도 `toxic_threshold=0.60` 변경 여부는 별도 승인으로 분리.
5. held-out 결과를 본 뒤 형식·후보 수·threshold·패턴을 조정하려는 시도가 발생하면
   보류/폐기하고 새 실험 카드로 분리한다.
