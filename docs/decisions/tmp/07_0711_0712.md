# 결정 카드 — 2026-07-11~12

## 평가 관측성·데이터셋 거버넌스

### D69. eval 진단은 실제 판정 실행의 trace를 사용
- **결정:** eval이 review 완료 후 검색을 재실행하지 않고, 실제 `review_contract` 안에서 발생한 rerank 후보를 프록시로 수집한다. 후보 없음은 NO_CANDIDATE/SEARCH_MISS로 기록한다.
- **이유:** 외부 리랭커 재실행 변동 때문에 최종 matched standard와 진단 top-3가 불일치하던 문제를 제거한다.
- **상태:** 구현·정합성 확인 (07-11)

### D70. v4는 독소 hard-negative challenge로 동결
- **결정:** v4의 단정적 법률 표현을 중립적 ‘검토 후보’ 문구로 고치고 스냅샷 테스트·해시로 동결한다. v4는 독소 held-out challenge이지 유형별 이탈 튜닝 근거가 아니다.
- **이유:** 법령 앵커가 법적 결론처럼 보이지 않게 하고, SI·SW에 이탈 양성이 없는 데이터 구성의 한계를 명시한다.
- **상태:** P0 완료 (07-12)

### D71. v5를 유형별 이탈 기준선과 독립 held-out으로 채택
- **결정:** SI/SM/SW 각 45건(총 135)의 v5와 topic-group split 격리를 구성하고, 유형별 이탈 성능 비교·독립 검증 기준선으로 동결한다.
- **이유:** v4의 유형별 양성 불균형을 해결하고, 오류 방향이 SI/SM UNDER_MATCH와 SW OVER_MATCH로 다름을 드러냈다. 전역 threshold 하나로 동시 해결하지 않는다.
- **상태:** 채택; 기본 threshold 유지 (07-12)

## 독소 개선의 실험 통제

### D72. 독소 오류를 재현 가능한 taxonomy로 분류
- **결정:** FP/FN을 SEARCH_MISS/LOW_SCORE/OVER_MATCH/UNMAPPED_PATTERN과 의미 축으로 분류한다.
- **이유:** 개선 대상을 점수 감각이 아닌 사례 단위의 결정론적 근거로 선택하기 위해서다. 당시 62건은 LOW_SCORE 26, OVER_MATCH 36으로 설명됐다.
- **상태:** 채택 (07-12)

### D74. 평가 CLI는 동결 산출물과 실험을 격리
- **결정:** `--case-ids`, `--no-write`, `--output-dir`로 split 선택·무기록 실행·별도 산출물 저장을 지원하고, trace는 동일 review 실행에서 수집한다.
- **이유:** 실험이 골든 산출물과 진단을 덮어쓰지 않게 하며, held-out 단회 실행 규칙을 지킨다.
- **상태:** 채택 (07-12)
