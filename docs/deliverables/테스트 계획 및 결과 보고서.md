# 테스트 계획 및 결과 보고서

## 1. 테스트 개요 및 목적

### 1.1 테스트 목적
- 본 테스트는 내외부 문서를 벡터 형태로 임베딩하여 구축한 RAG(Retrieval-Augmented Generation) 기반 시스템의 파이프라인별 성능을 정량적으로 검증하기 위함입니다.
- **"RAG에서 답변이 이상하면 대부분 LLM 문제가 아니라 검색 문제가 먼저"**라는 가설을 검증하기 위해 검색 알고리즘별 Ablation 성능을 비교 평가합니다.
- 무조건 답변을 많이 생성하는 것이 아니라, **"근거가 없을 때 답변을 제한(환각 방지)"**하고, 표준 대비 변경점(이탈) 및 독소 조항을 정확하게 걸러내는지 오차 행렬 기반 지표로 측정합니다.

### 1.2 테스트 목표
- **정성적 목표:** "검색만으로 표준 대비 이탈을 탐지할 수 있다"는 명제 검증 및 `NO_MATCH` 처리와 근거 원문 첨부 메커니즘의 정상 작동 확인
- **정량적 목표:** BM25 / Dense / Hybrid / Hybrid + Reranker의 단계별 비교를 통해 RAG 아키텍처 도입의 필요성을 수치로 증명

---

## 2. 테스트 계획 및 설계

### 2.1 대상 데이터 및 함정 케이스 설계
- **평가 데이터:** 사람이 직접 라벨링한 합성 계약서 및 실제 샘플 계약서 기반의 골든 데이터셋 (총 3개 유형, n=92)
- **리스크 변이(함정 케이스):** 시스템의 강건성을 시험하기 위해 의도적으로 **[말바꿈 / 조항 순서 뒤바꿈 / 부분 누락]** 변이를 골든셋에 포함하여 설계

### 2.2 평가 트랙 및 지표 (Metrics)
1. **[Track A-1] 검색(Retrieval) Ablation:** Recall@5 (5개 후보 내 정답 포함 확률), MRR (Mean Reciprocal Rank)
2. **[Track A-2/A-3] 이탈·독소 조항 분류:** 정밀도(Precision), 재현율(Recall), F1-Score (참음성(TN) 지표 포함 산출)
3. **[Track B] 실계약 커버리지 검증:** 실제 프리랜서 계약서 5건을 투입하여 표준 유형(SW_FREELANCE)에 대한 일관된 정합성 및 커버리지($\frac{\text{표준 조항} - \text{MISSING}}{\text{표준 조항}}$) 측정

## 3. 테스트 결과 및 분석 (Test Results)

### 3.1 [A-1] 검색 방식별 성능 비교 (Ablation)
검색 컴포넌트 조합에 따른 RAG 파이프라인의 핵심 성능 지표입니다.

| 방식 (Variant) | Recall@5 | MRR |
| :--- | :---: | :---: |
| **BM25 (키워드)** | 88.4% | 0.804 |
| **Dense (임베딩)** | 93.0% | 0.844 |
| **Hybrid** | 93.0% | 0.829 |
| **Hybrid + Reranker (최종)** | **94.2%** | **0.874** |

- **분석 결과:** 전통적인 키워드 방식(BM25) 대비 `Hybrid + Reranker` 최종 모델 적용 시 **Recall@5 기준 +5.8%p, MRR 기준 +0.070의 명확한 성능 개선**이 확인되었습니다. 의도적인 문구 변형(말바꿈, 누락) 함정 케이스 속에서도 최적의 근거 조항을 상위에 노출하는 능력을 확보했습니다.

---

### 3.2 [A-2 / A-3] 계약 유형별 이탈·독소 조항 분류 성능
`difflib` 및 정규식을 활용하여 3개 계약 유형(총 n=92)의 리스크를 분류한 결과입니다.

| 계약 유형 | 분류 | Precision | Recall | F1-Score |
| :--- | :---: | :---: | :---: | :---: |
| **SW_FREELANCE**<br>(n=17) | 이탈<br>독소 | 1.000<br>0.545 | 0.154<br>1.000 | 0.267<br>**0.706** |
| **SI_SUBCONTRACT**<br>(n=25) | 이탈<br>독소 | 0.750<br>0.750 | 0.200<br>0.429 | 0.316<br>0.545 |
| **SM_SUBCONTRACT**<br>(n=50) | 이탈<br>독소 | 0.500<br>0.500 | 0.032<br>0.077 | 0.061<br>0.133 |

- **분석 결과:** - `SW_FREELANCE` 유형의 독소 조항 탐지 성능이 **F1-Score 0.706 (Recall 1.000)**으로 가장 우수하게 작동하고 있습니다. 위험 조항을 놓치지 않고 잡아내는 방어선 역할을 성공적으로 수행 중입니다.
  - 반면, 전체적으로 **이탈(변경점) 탐지의 재현율(Recall)이 낮게 측정**되는 경향을 보입니다. 특히 `SM_SUBCONTRACT` 유형의 경우 탐지율이 크게 저조하여, 규칙 기반 드라이버의 정교화가 시급한 개선 과제로 식별되었습니다.

---

### 3.3 [Track B] 실제 프리랜서 계약서 5건 — 표준유형 커버리지 검증
형태가 다른 5종의 실제 계약서를 표준 프리랜서 계약서(SW_FREELANCE) 템플릿과 매칭하여 커버리지를 테스트한 결과입니다.

| 투입 문서 | 조항 수 | SW_FREELANCE | SI_SUBCONTRACT | SM_SUBCONTRACT |
| :--- | :---: | :---: | :---: | :---: |
| **위탁 업무 계약서** (일반 위탁업무) | 14 | **52%** | 11% | 11% |
| **웹 개발 용역 계약서** (IT·개발) | 15 | **52%** | 10% | 11% |
| **프리랜서 고용계약서 샘플 1** | 12 | **43%** | 8% | 7% |
| **프리랜서 고용계약서 샘플 2** | 10 | **35%** | 7% | 6% |
| **프리랜서 고용계약서 샘플 3** | 14 | **52%** | 10% | 9% |

- **분석 결과:** 테스트를 진행한 5건의 실제 계약서 모두 타 유형 대비 `SW_FREELANCE` 표준 유형과 매칭률이 압도적으로 높게 나타나 **Best-fit 판정 일관성을 완벽하게 입증**했습니다. 또한, 평가 과정에서 `NO_MATCH`가 0건 발생하여 변형된 실무 문서 환경에서도 전 조항 매칭을 안정적으로 성공시켰습니다.

## 4. 종합 결론 및 향후 개선 과제

1. **RAG 파이프라인의 기술적 정당성 확보:** `Hybrid + Reranker` 아키텍처를 통해 최종 검색 성능(Recall@5 94.2%)을 확보함으로써, 향후 LangChain과 LLM 연동 시 오답변을 원천 차단할 수 있는 강력한 전단 엔진을 완성했습니다.
2. **분류 재현율(Recall) 불균형 개선 필요:** 독소 조항 분류 성능 대비 이탈 조항 분류 지표(F1 및 Recall)가 전반적으로 낮습니다. 이는 현재의 결정론적 규칙 표현식(`re`)이 실제 계약서의 유연한 문맥 변형(말바꿈, 조항 결합 등)을 지나치게 엄격하게 거르고 있음을 뜻합니다.
3. **향후 반영 사항 (Next Action):** 차기 버전에서는 `difflib` 유사도 임계값 기준을 유연하게 보완하고, 성능이 저조한 SM/SI 유지보수 도메인 전용 정규식 패턴 세트를 보강하여 탐지 성능 개선에 집중할 예정입니다.
